Get Started

The Darwin RESTful API is a community driven project to build a RESTful interface for the the Darwin Webservice - the code is open source, and available to any Trainhacker. You can get started by visiting the documentation. Remember to get your Darwin API Key! If you have any questions, please feel free to contact us at hello@hackpartners.com.




Big Data, Darwin, and the Railway Industry.

Darwin is one of the key customer facing datafeeds used by the railway industry. It provides real-time train movement data to all franchised passenger train operating companies in the UK, as well as a host of other railway service providers. These datafeeds enable them to deliver high quality information about the train services that get us from A to B every day.



Innovating. One Dataset at a time.

The HackTrain kick-started the Darwin RESTful project - an unofficial Darwin REST interface supported by the HackTrain team. The vision behind this project is to enable all developers, including those unfamiliar with rail, to use and interpret the data from Darwin through a RESTful interface that provides data that in a common JSON format. We aim to enable more trainhackers to capitalise on Darwin data and build unique, innovative projects that could revolutionise the rail industry.

We are also working with in other projects using the darwin interface. This includes our project with the Darwin Pushport interface. You can view the documentation here.






Built by the community, for the community.

The Darwin RESTful API is a community driven project - the code is open source, and available for any Trainhacker. Pull requests are more than welcome. We encourage trainhackers to contribute, as we look for this project to become as tailored as possible for the community itself.
